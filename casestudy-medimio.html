Designing an accessible, engaging learning platform for elementary school students and teachers—where education meets intuitive digital experiences.

Project Overview

MediMio is a comprehensive digital learning platform designed specifically for elementary school education. The platform serves two distinct user groups: young students (ages 6-10) who need an engaging, intuitive interface, and teachers who require powerful tools to manage curriculum, track progress, and support individual learning paths.

The challenge was to create a single platform that could serve both audiences effectively—making learning fun and accessible for children while providing teachers with the sophisticated tools they need to manage their classroom effectively.

The Challenge

Dual User Groups

Designing for both elementary school students (ages 6-10) and their teachers, each with vastly different needs and technical abilities.

Engagement vs. Learning

Creating an interface that's playful enough to engage young learners while maintaining educational integrity and not distracting from learning goals.

Accessibility First

Ensuring the platform works for all students, including those with learning differences, motor skill challenges, and varying reading levels.

Research & Discovery

Understanding Young Users

We conducted observational studies in three elementary schools, watching how students interact with existing educational technology. Key insights: children respond to immediate visual feedback, get frustrated with tiny touch targets, and learn better with clear visual hierarchies rather than text-heavy instructions.

Teacher Interviews

15 teachers shared their pain points with current learning management systems: too much time spent on admin tasks, difficulty tracking individual student progress, and platforms that don't accommodate different learning styles. They needed efficiency without sacrificing educational quality.

Accessibility Audit

We consulted with special education specialists to ensure our design would work for students with varying abilities—from those with dyslexia to students with motor skill challenges. This wasn't an afterthought; it shaped our core design decisions.

Design Solutions

Age-Appropriate Interface Design

Large, colorful buttons and icons that are easy for small hands to tap. Clear visual hierarchy with minimal text, using illustrations and icons to convey information quickly.

Teacher Dashboard

Separate, sophisticated interface for teachers with progress tracking, assignment management, and detailed analytics—without overwhelming complexity.

Progressive Complexity

Interface that adapts as students advance, starting with simple layouts and gradually introducing more features as they become comfortable with the platform.

Universal Design Principles

High contrast text, screen reader support, keyboard navigation, and alternatives for every interaction method. Font sizing options and dyslexia-friendly typography.

Design Process

1

Wireframing

Started with low-fidelity wireframes to establish information architecture for both student and teacher interfaces. Iterated rapidly based on feedback from educators and usability testing with children.

2

Visual Design

Developed a vibrant, child-friendly color palette while maintaining high contrast for readability. Created a library of friendly illustrations and icons that communicate without relying heavily on text.

3

Interactive Prototyping

Built clickable prototypes to test navigation flows and interaction patterns. Tested with actual students to observe pain points and moments of confusion—then refined accordingly.

4

Accessibility Testing

Conducted thorough testing with screen readers, keyboard navigation, and various assistive technologies. Made continuous refinements to ensure WCAG AA compliance and beyond.

5

User Testing

Multiple rounds of testing with both teachers and students. Observed real classroom implementation to identify issues that only emerge in authentic educational contexts.

Key Features

Student Learning Hub

  • Colorful, gamified interface with achievement badges
  • Large, tap-friendly buttons and icons
  • Visual progress indicators
  • Audio instructions for pre-readers
  • Adjustable text size and dyslexia-friendly fonts

Teacher Dashboard

  • Class-wide and individual student progress tracking
  • Quick assignment creation and distribution
  • Real-time engagement analytics
  • Customizable learning paths per student
  • Export reports for parent-teacher conferences

Interactive Learning Modules

  • Multimedia content (video, audio, interactive elements)
  • Adaptive difficulty based on student performance
  • Immediate feedback on exercises
  • Collaborative activities for group learning
  • Offline mode for areas with limited connectivity

Accessibility Tools

  • Full screen reader support
  • Keyboard-only navigation
  • High contrast mode
  • Configurable color schemes
  • Alternative text for all visual elements

Key Learnings

Designing for Children Requires Restraint

It's tempting to make everything colorful and animated for kids, but we learned that too much stimulation becomes distracting. The most effective design was clean, focused, and used animation purposefully to guide attention rather than entertain.

Accessibility Benefits Everyone

Features we implemented for accessibility—like clear visual hierarchy, large touch targets, and audio alternatives—ended up improving the experience for all users, not just those with disabilities.

Test in Real Contexts

Lab testing only revealed so much. Observing the platform being used in actual classrooms—with 25 students, varying noise levels, and real-world constraints—uncovered issues we never would have found otherwise.
